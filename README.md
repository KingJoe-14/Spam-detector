📩 SMS Spam Detection
A machine learning project that classifies SMS text messages as spam or ham (legitimate), built from scratch with scikit-learn and wrapped in a Django web app for live predictions.

Overview
This project walks through the full lifecycle of a text classification model:

Pulling the dataset directly from Kaggle
Cleaning and exploring the text
Converting text into numerical features with TF-IDF
Training and comparing three classic models
Selecting the best-performing one
Serving it through a Django web app with a live text box demo
Dataset
Source: SMS Spam Collection Dataset (Kaggle), originally from the UCI Machine Learning Repository
Size: 5,572 messages (5,169 after removing duplicates)
Labels: ham (~87%) / spam (~13%)
Pipeline
Step	What happens
Load	Read spam.csv, keep only the label + message columns
Clean	Lowercase, strip URLs/punctuation, collapse whitespace
Split	80% train / 20% test, stratified to preserve class balance
Vectorize	TfidfVectorizer — unigrams + bigrams, English stop words removed
Train	Multinomial Naive Bayes, Logistic Regression, Linear SVM
Evaluate	Precision, recall, F1 on the held-out test set
Save	Best model + vectorizer serialized with joblib
Serve	Django app loads the model and predicts on user input
Results
Evaluated on 1,034 held-out test messages:

Model	Precision	Recall	F1
Naive Bayes	1.00	0.76	0.86
Logistic Regression	0.93	0.90	0.91
Linear SVM (selected)	0.97	0.87	0.92
Linear SVM was selected as the best model based on F1 score, balancing high precision (rarely flags a real message as spam) with solid recall (catches most actual spam).

class_weight="balanced" was used for Logistic Regression and SVM to account for the ~87/13 class imbalance in the dataset.

Getting started
1. Train the model (notebook)
Open notebooks/spam_detection_stepbystep.ipynb in Jupyter. It walks through downloading the dataset from Kaggle (requires a free Kaggle account + API key), cleaning the data, training all three models, and saving the best one as spam_classifier.joblib + tfidf_vectorizer.joblib.

2. Run the web demo (Django)
bash
cd spamdetector
pip install -r requirements.txt
# copy spam_classifier.joblib and tfidf_vectorizer.joblib into this folder
python manage.py runserver
Then open http://localhost:8000 and paste a message in to see a live SPAM / HAM prediction with a confidence score.

Tech stack
Python, pandas — data loading and cleaning
scikit-learn — TF-IDF vectorization, model training, evaluation
joblib — model serialization
Django — web app serving live predictions
Known limitations
The dataset is SMS text from ~2011–2012 — it reflects that era's spam patterns (prize scams, premium-rate call-backs) and may miss newer styles like phishing links or "urgent account verification" scams that don't share the same vocabulary.
The model works on raw word patterns, not URLs, sender metadata, or message context, since none of that is present in the dataset.
Possible next steps
Add phishing-style examples to improve coverage of newer scam patterns
Try character-level n-grams to catch obfuscated spelling (e.g. "fr33")
Visualize which words most influence spam/ham predictions
Deploy the Django app to a public host
